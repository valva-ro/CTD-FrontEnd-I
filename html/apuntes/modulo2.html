HTML

Hyper Text Markup Language: es un formato de texto electrónico que puede viajar a través de una red de manera rápida y fácil, que además puede ser interpretado con facilidad por un navegador web.

Cada elemento dentro de un documento HTML representa un componente visual en el navegador.

Estructura básica

xxxxxxxxxx
<!DOCTYPE html>                     <!-- version de html -->
<html>                              <!-- etiqueta raiz -->
    <head>                          <!-- configuracion de la pag, comunicacion con el navegador -->
        <meta charset="utf-8">      <!-- codificacion de caracteres -->
        <title>Hola mundo</title>   <!-- titulo de la pestaña -->
    </head>                 
    <body>                          <!-- contenido de la pag, comunicacion con el usuario -->       
        =
    </body>
</html>

Buenas Prácticas

Semántica

Utilizar etiquetas semánticas nos permite generar un código más entendible, tanto para los humanos como para los buscadores.

  • <head></head> para información sobre el documento
  • <body></body> para el contenido
  • <header>: cabecera de documento o sección. En el caso de ser el header del documento lo usual es colocar el logotipo, la barra de navegación, los enlaces a las redes sociales y algún campo de búsquedas rápidas
  • <nav>: barra de navegación. Dentro se suelen colocar listas desordenadas, elementos de listas y enlaces
  • <footer>: pie de página del documento o de una sección. Tradicionalmente van cosas como los derechos reservados y algunos enlaces adicionales
  • <section>: una sección de contenido
  • <article>: una pieza de contenido independiente, podría funcionar por sí solo como un producto, servicio, noticia, etc. Al ser independiente puede tener dentro su propio nav, main, body, section, footer, etc. En general los article van dentro de secciones aunque no es obligatorio.

Indentación

La indentación se utiliza para mejorar la legibilidad del código y comprender qué estructura está dentro de otra en términos de código. Se realiza mediante un espacio al principio de la línea de código.

Display: bloque o linea

Bloque

Ocupan el 100% del ancho del sitio, visualmente generan un salto de línea.

Por ej: <div>

Línea

Determinan su tamaño en base al contenido que tengan, Si el contenido excede la línea, continúa en la de abajo.

Por ej: <span>

CSS

  • inline: define un elemento con comportamiento en línea. No recibe propiedades del modelo de caja
  • block: define un elemento con comportamiento de bloque. Puede recibir propiedades del modelo de caja
  • inline-block: define un elemento con comportamiento de semi-bloque. Puede recibir propiedades del modelo de caja y también comparte propiedades de elementos de línea
  • none: oculta un elemento

Etiquetas

  • <div></div> división dentro del contenido

  • <a></a> para enlaces

     

  • <h1></h1>…<h6></h6> para títulos dentro del contenido

  • <img> para añadir imágenes al documento

  • Listas:

    • <ol></ol> para listas ordenadas
    • <ul></ul> para listas desordenadas
    • <li></li> para elementos dentro de la lista

  • <p></p> para parágrafos

  • <span></span> para estilos de una parte del texto

Ruta

Una ruta es una dirección o camino que le va a permitir al navegador encontrar un recurso (que puede ser una página web, una imagen, un video, etc).

¿De qué depende si usamos una ruta absoluta o relativa? De si la ruta está dentro o fuera de nuestro sitio. En el caso de una ruta que está fuera de nuestro sitio decimos que es un recurso externo y utilizamos una ruta absoluta, mientras que si esta dentro decimos que es un recurso local y utilizamos una ruta relativa.

Además de las rutas, podemos hacer referencias a fragmentos o elementos dentro de la misma página con las anclas que sería básicamente poner el id de un elemento como atributo del href.

CSS

La Cascading Style Sheet (Hoja de Estilos en Cascada) nos sirven para estilizar nuestro contenido HTML. Con CSS podemos cambiar colores, fondos, tipografías, acnchos, altos y mucho más.

Vinculando HTML y CSS

Existen tres formas para vincular un CSS a un archivo HTML

Vinculación externa (recomendada)

Es la más utilizada ya que podemos escribir todos nuestros estilos e un archivo CSS y vincularlo al HTML usando la etiqueta <link> dentro del <head>.

Vinculación interna (no recomendada)

A través de la etiqueta <style> dentro del <head>

Vinculación en línea (no recomendada)

Usando el atributo style en cada elemento de nuestro HTML

Reglas

Una regla de CSS es un conjunto de instrucciones que se aplican a un elemento determinado para agregarle estilos.

Primero debemos escribir el o lo selectores, seguidos de las llaves {} que indican el comienzo de las reglas. Cada regla se compone de una propiedad y su correspondiente valor separados por dos puntos :. Finalmente podemos tener mnás de una regla por elemento, separadas por punto y coma ;.

xxxxxxxxxx
body {
    background-color: black;
    font-family: sans-serif;
    text-align: center;
}

Propiedades

Existen muchas propiedades de CSS que nos permiten manipular los elementos del HTML a nuestro antojo. Si tuviéramos que agrupar las propiedades los grupos quedarían:

Selectores

De ID

Este selector va a atrapar al elemento HTML que tenga asignado el atributo id con el valor correspondiente. Se recomienda utilizar nombres únicos para cada elemento y no repetirlos a lo largo del documento.

xxxxxxxxxx
<!-- HTML -->
<h3 id="saludo">Hola!</h3>
xxxxxxxxxx
/* CSS */
#saludo {
    color: blue
}

De clase

Este selector va a atrapar al elemento HTML que tenga asignado el atributo class con el valor correspondiente. Podemos asignarle la cantidad de clases que queramos a un mismo elemento simplemente separándolas con un espacio.

xxxxxxxxxx
<!-- HTML -->
<h3 class="noticia destacada">Esta es una noticia!</h3>
xxxxxxxxxx
/* CSS */
.noticia {
    font-size: 22px;
}
.destacada {
    color: blue
}

De etiqueta

Este selector va a atrapar al elemento HTML con el mismo nombre de etiqueta que llamemos desde nuestro CSS.

xxxxxxxxxx
<!-- HTML -->
<p>
    Lorem ipsum dolor sit amet, consectetur adipiscing elit. Etiam tempor lacinia justo, a egestas mauris volutpat in.</p>
xxxxxxxxxx
/* CSS */
p {
    color: gray
}

Combinados

Como su nombre lo indica, son la combinación de algunos o todos los anteriores. En el siguiente ejemplo combinaremos un selector de etiqueta con uno de clase.

xxxxxxxxxx
<!-- HTML -->
<h2 id="subtitulo">Un subtítulo</h2>
xxxxxxxxxx
/* CSS */
h2.subtitulo {
    color: yellow
}

Descendentes

Sirven para agregar especificidad. En el ejemplo vamos a atrapar al elemento li que está dentro del ul con el id lista. CSS siempre va a priorizar los selectores más específicos para aplicar los estilos.

xxxxxxxxxx
<!-- HTML -->
<ul id="lista">
    <li>Primer elemento</li>
</ul>
xxxxxxxxxx
/* CSS */
ul#lista li {
    text-align: center
}

Fuentes

Para que la font-family elegida funcione, debe existir en la computadora del usuario o usar una webfont. Podemos poner más de una tipografía separando las adicionales por comas. En caso de que la primera no esté disponible se cargará la segunda, y así sucesivamente.

xxxxxxxxxx
p {
    font-family: Arial, sans-serif;
}

Podemos definir el tamaño tipográfico con font-size. Este atributo recibirá un valor numérico acompañado de la unidad de medida. Las más habituales serán px, em y rem.

xxxxxxxxxx
p {
    font-size: 20px;
}

Con font-style podemos definir el estilo de la tipografía. Recibe los valores italic, normal y oblique.

xxxxxxxxxx
p {
    font-style: normal;
}

Con font-weight podemos definir el peso de la tipografía. Recibe los valores bold, lighter, normal, entre otros. También puede recibir un valor numérico que se irá incrementando de 100 en 100.

xxxxxxxxxx
p {
    font-weight: 500;
}

Con text-align podemos definir la alineación del texto. Los valores que recibe son center, left, right, inerit y justify. El valor por defecto para todos los elementos es left.

xxxxxxxxxx
p {
    text-align: justify;
}

Con text-decoration vamos a poder definir un tipo de decoración para el texto. Recibe los valores line-through, underline, overline y none.

xxxxxxxxxx
p {
    text-decoration: underline;
}

Con line-height vamos a poder definir el alto de cada línea de textos (o interlineado). Recibe un valor numérico acompañado de la unidad de medida.

xxxxxxxxxx
p {
    line-height: 20px;
}

Fondos

Como fondo podemos poner un color o una imagen. Para eso, utilizaremos la propiedad background-color o background-image.

Importante: si decidimos utilizar un color y una imagen, esta última estará sobre el color, así que es posible que este no se vea.

xxxxxxxxxx
/* cualquier notación funciona */
body {
    background-color: #ffffff;
    background-color: black;
    background-color: rgb(0,0,0);
}
xxxxxxxxxx
body {
    background-image: url("../img/imagen.png");
    /* 
        los valores posibles son repeat, no-repeat, repeat-x o repeat-y 
    */
    background-repeat: no-repeat;
    /* 
        nos permite mover la imagen dentro del elemento 
        puede tener 1 o 2 valores, el primero eje x, el segundo eje y
    */
    background-position: right top;
    /* 
        nos permite definir si la imagen de fondo se movera junto con la página o si
        quedará fija
        los valores posibles son fixed, scroll, inherit e initial
    */
    background-attachment: fixed;
    /* 
        nos permite definir el tamaño de la imagen
    */
    background-size: 100px;
}

Color

También podemos usar la notación rgba que nos va a permitir indicar la transparencia que queramos que tenga agregando un atributo más que irá entre 0 y 1 (donde 0 es totalmente transparente y 1 totalmente visible). Por ej: rgba(255, 100, 50, 0.5).

Otra opción es usar los nombres (ver [acá(https://htmlcolorcodes.com/color-names/)]).

Iconos

El atributo principal de los iconos es proporcionar un lenguaje visual común que una brechas lingüísticas. Son intuitivamente reconocibles y no tienen una interpretación abierta, es justamente esto lo que los hace perfectos para un contexto digital, en el que interactúan muchas personas de culturas diferentes.

Al igual que como ocurre con las fonts web, es necesario, en primer lugar, seleccionar la librería de iconos que vamos a utilizar.

Hay muchísimas librerías gratuitas y también de pago disponibles en la web. Estas son algunas de ellas:

  • Font Awesome
  • Iconic
  • Material Design

Font Awesome es una librería como cualquier otra, por tanto, para usarla lo primero que debemos hacer es incluir esa librería —generalmente al inicio del documento—, o bien tener en nuestros servidor los archivos de fuente con que trabaja FontAwesome.

La forma más simple sería incluir la propia librería de ellos. Los pasos para hacerlo son:

  1. Incluimos el enlace a nuestros iconos dentro del <head> del HTML: <link rel="stylesheet" href="https://use.fontawesome.com/releases/v5.15.2/css/all.css">
  2. Insertar el icono seleccionado en el lugar del <body> que más nos guste: <i class="fas fa-exclamation-triangle"></i>

Recordemos que también es posible tenerlos alojados en nuestro servidor. En ese caso, primero tenemos que descargarlos y, posteriormente linkear en nuestro html el CSS y el JS correspondiente.

Fuentes

Fuentes seguras

Es el conjunto de familias tipográficas que pueden ser usadas en cualquier página web sin problemas porque están preinstaladas en todos los SO.

¿Qué problemas podemos tener con las fuentes?

Fuentes genéricas

Si queremos tener todo perfectamente controlado la lista de tipografías seria interminable, entonces ¿cómo podemos asegurarnos de que nuestro sitio se vea bien sin complicarnos?

Las familias genéricas son:

  • serif
  • san-serif
  • monospace
  • cursive
  • fantasy

Fuentes web

Las fuentes web no siempre están preinstaladas en un dispositivo y, por lo tanto, deben ser descargadas en el navegador del usuario antes de ser mostradas. Algunos ejemplos de fuentes web incluyen la fuente Open Sans y Roboto de Google, así como Acumin o Cortado de Adobe Fonts.

Al utilizar fuentes de terceros, como Google o Adobe Fonts, solo estamos incluyendo las fuentes mediante la vinculación a un activo externo. Alojar fuentes localmente significa que tenemos los archivos de fuentes en nuestro propio servidor. Para ello vamos a hacer uso de la directiva css @font-face.

La regla @font-face permite cargar fuentes personalizadas en una página web. Una vez que se agrega a una hoja de estilo, la regla indica al navegador que descargue la fuente desde donde está alojada y luego la muestre como se especifica en el CSS.

Modelo de caja

Cada elemento en TML es una caja compuesta de márgenes (margin), bordes (border), relleno (padding) y finalmente contenido (content). Con CSS podemos manipular todas estas propiedades para cambiar la apariencia de cada elemento. A su vez, estas propiedades pueden aplicarse de manera diferente a los cuatro lados de la caja (top, bottom, left, right).

Padding

Podemos asignarle 1, 2, 3 o 4 valores.

xxxxxxxxxx
padding: 12px;                    /* se aplica 12px a todos los lados */
padding: 12px, 20px;              /* 12px arriba y abajo, 20px izquierda y derecha */
padding: 12px, 20px, 16px;        /* 12px arriba, 20px izquierda y derecha, 16px abajo */
padding: 12px, 20px, 16px, 14px;  /* 12px arriba, 20px derecha, 16px abajo, 14px izquierda */

Border

Podemos asignarle el estilo de línea, espesor y color:

  • Estilo: solid, dotted, dashed o double
  • Espesor: cualquier unidad de medida
  • Color: cualquier color en cualquier notación válida de CSS

Margin

Es el margen exterior del elemento, nos sirve para separar una caja de otra. Al igual que el padding podemos asigarnle entre 1 y 4 valores.

Box sizing

Por defecto el valor será content-box y el comportamiento será el de aplicarle el ancho y alto que definamos al conenido del elemento.

Si le asignamos el valor border-box el ancho y alto que indiquemos tomará en cuenta no solo el contenido del elemento, sino también el padding y el borde, dejando solo el margen por fuera.

Es muy común asignarle box-sizing: border-box a todos los elementos del sitio.

Width y Height

De manera predeterminada el ancho (width) de un elemento será el 100% de su padre contenedor, pero podemos asignarle un valor nosotros usando cualquier unidad de medida.

Por su parte la altura (height) predeterminada de un elemento será la que su contenido le proporcione, ya sea de bloque o de línea. Al igual que el ancho, podemos asignarle un valor usando cualquier unidad de medida. En general no se recomienda usa porcentajes.

Posicionamiento

  • relative: se desplaza el elemento desde su posición original sin modificar el flujo original de los demás elementos.

  • absolute: se desplaza el elemento desde el body. Modifica el flujo original de los demás elementos, ya que al moverse ese espacio "queda vacío" y otros elementos lo ocupan.

  • relative + absolute: si nuestras cajas hijas están dentro de otra caja padre, el punto de referencia seguirá siendo el body a menos que el posicionamiento del padre del fuera relative.

  • fixed: desplaza el elemento tomando como referencia la ventana del navegador. El elemento siempre se mantendrá fijo con respecto a los costados de la ventana.

  • sticky: es considerado un híbrido entre relativo y dijo ya que es tratado como un elemento relativamente asta que cruza un umbral especificado, en cuyo punto se trata como fijo asta que alcanza el límite de su padre.

Z-Index

Esta propiedad controla como se apilan las capas en CSS, pero solo podremos modificárselo a los elementos que tengan position como relative, absolute o fixed.

Supongamos que tenemos el siguiente código dentro del body:

xxxxxxxxxx
<div class="caja-1">1</div>
<div class="caja-2">2</div>
<div class="caja-3">3</div>

Y que visualmente se ve así:

Si modificamos el CSS agregándole lo siguiente:

xxxxxxxxxx
.caja-1 {
    ...
    position: relative;
    left: 100px;
    top: 50px;
}

El z-index del elemento será 0, y se mostrará por encima de los otros

Si agregamos también lo siguiente: 

xxxxxxxxxx
.caja-1 {...}
.caja-2 {
    position: relative;
    z-index: 10;
}

La segunda caja se moverá delante del resto porque ahora tiene mayor valor.
