Diseño adaptativo

Viewports

En palabras sencillas, el viewport es la parte visible de cualquier navegador web, en la cual se puede ver el contenido de la página, sin tener que hacer scroll.

La etiqueta <meta> viewport da al browser instrucciones de cómo se debe dimensionar y escalar la página web al cargarse. 

xxxxxxxxxx
<meta name="viewport" content="width=device-width, initial-scale=1">

Medidas relativas

Las medidas relativas son aquellas que tienen en cuenta el contexto donde se encuentran. Si el contexto cambia, estas medidas cambiarán con él. Según de qué medida estemos hablando, el número que pongamos será relativo a:

  • El contenedor padre
  • El tamaño de la fuente del sitio
  • El tamaño de la fuente del contenedor padre
  • El tamaño del viewport

Si tomamos los porcentajes como ejemplo, podemos decir que son una unidad relativa, puesto que 30% de ancho no será lo mismo para un elemento situado dentro de un contenedor de 2000px de ancho que para un contenedor de 1000px de ancho.

Porcentajes

Cualquier medida expresada en porcentaje siempre estará relacionada con la medida (en ese mismo eje) del elemento padre que la contiene. Si el contenedor padre mide 300px de ancho y le asignamos un ancho del 50% al elemento interior, este medirá 150px (el 50% del ancho padre).

No se recomienda usar porcentajes para el alto de un elemento.

Para calcular los porcentajes vamos a usar la regla de 3 simple:

em y rem

Los ems son siempre relativos al elemento padre. Tomarán como valor de referencia la propiedad font-size. El punto de referencia inicial es el valor de font-size del elemento <html> que por defecto es 16px. El resto de los elementos tendrán 1em de font-size que equivale al mismo valor que tenga el padre.

Si le asignamos 1.5em al font-size de cualquier elemento, el tamaño resultante será el valor del padre multiplicado por el valor en ems → 16px * 1.5 = 24px

Si utilizamos em en una propiedad que no sea font-size, se tomará para el cálculo el font-size que tenga dicho elemento. Por ejemplo:

xxxxxxxxxx
p {
    font-size: 20px;
    line-height: 2em;   /* 20px * 2 = 40px */
    padding: 1.5em;     /* 20px * 1.5 = 30px */
}

Usar ems puede ser muy complicado, por eso es que lo recomendable es usar rems que funcionan de manera similar, con la diferencia de que siempre tomarán de base el tamaño font-size del elemento <html>.

De está manera el tamaño expresado en rems no modificará el de los elementos hijos y tampoco se verá afectado por el padre.

vw y vh

  • vw → Viewport Width es relativo al ancho total del viewport
  • vh → Viewport Height es relativo al alto total del viewport

Ambas medidas están expresadas como porcentajes del total, entonces si escribimos 50vh será el equivalente al 50% del alto disponible en el viewport

xxxxxxxxxx
div {
    width: 25vw;    /* 25% del ancho disponible */
    height: 50vh;   /* 50% del alto disponible */
}

Media queries

Son un conjunto de reglas de CSS que nos permiten cambiar los estilos de los elementos en función de las características del dispositivo que esté visualizando nuestro sitio.

La mayor parte de nuestros estilos estará fuera de las media queries. Dentro de cada media query escribiremos solo aquello que necesitemos ajustar para ese tamaño de viewport. Por lo general, se escriben al final de nuestra hoja de CSS.

min-width

Al especificar esto le estamos diciendo → "si como mínimo el viewport tiene n pixeles de ancho, apliquemos estas reglas"

xxxxxxxxxx
@media (min-width: 460px){
    body {
        background: red;
    }
}

max-width

Al especificar esto le estamos diciendo → "si como máximo el viewport tiene n pixeles de ancho, apliquemos estas reglas"

xxxxxxxxxx
@media (min-width: 768px){
    body {
        background: yellow;
    }
}

Orientación

Al especificar la orientación (portrait o landscape) estamos diciendo → “si como máximo el viewport tiene N píxeles de ancho y además el dispositivo está en posición vertical/horizontal, apliquemos estas reglas”.

xxxxxxxxxx
@media (max-width: 460px) and (orientation: landscape){
    body {
        background: yellow;
    }
}

Breakpoints

Estrategias de diseño

Mobile first

Si utilizamos mobile first como estrategia de diseño, la idea es determinar de manera general las reglas CSS para pequeñas pantallas para luego, a través de media queries, ir aclarando el comportamiento en viewports más grandes.

xxxxxxxxxx
body {
    background: red;
}
@media (min-width: 460px){
    /* Tablets */
}
@media (min-width: 768px){
    /* Laptop */
}

Mobile last

Si utilizamos mobile last como estrategia de diseño, la idea es determinar de manera general las reglas CSS para grandes pantallas para luego, a través de media queries, ir aclarando el comportamiento en viewports más pequeños.

xxxxxxxxxx
body {
    background: red;
}
@media (min-width: 460px){
    /* Tablets */
}
@media (min-width: 768px){
    /* Laptop */
}

Flexbox

Es una metodología de CSS que permite maquetar un sitio web utilizando una estructura de filas y columnas. Propone un único flujo en el que dispondremos de los elementos con mayor libertad para distribuir, redimensionar y reordenar cada uno de ellos en función de dicho flujo.

Ejes

Un eje es aquella representación imaginaria a partir de las líneas que definen la dirección frente a la cual se desplaza un elemento. En un contenedor flex los ejes van a definir la orientación a partir de la cual se desplazarán los elementos internos del mismo.

Hay dos ejes: el principal llamado main axis y el transversal llamado cross axis. Ninguno etá definido por el plano horizontal ni vertical, sino que dependerá de la configuración que nosotros dispongamos.

flex-direction

Nos sirve para definir cual va a ser el main axis del contenedor (el cross axis será el perpendicular al que tenga main).

  • row 🠖 los items se disponen en el eje x de izquierda a derecha. Es el valor que toma flex por defecto

  • row-reverse 🠖 los ítems se disponen en el eje x de derecha a izquierda

  • column 🠖 los ítems se disponen en el eje y, de arriba hacia abajo

  • column-reverse 🠖 los ítems se disponen en el eje y, de abajo hacia arriba

Alinear con flexbox

  • justify-content 🠖 para el main axis

    • flex-start 🠖 es el valor por defecto, los elementos se alinean con respecto al inicio del eje

    • flex-end 🠖 los elementos se alinean con respecto al final del eje

    • center 🠖 los elementos se alinean en el centro del eje

    • space-between 🠖 los elementos se distribuyen de manera uniforme. El primer elemento ira al inicio, el último al final y el resto se reparten dejando el mismo espacio entre cada uno

    • space-around 🠖 los elementos se distribuyen de manera uniforme. La diferencia con el anterior, es que el espacio disponible se reparte entre todos los elementos de la misma manera

  • align-items 🠖 para el cross axis

    • stretch 🠖 los elementos se ajustan para abarcar todo el contenedor

    • flex-start 🠖 los elementos se alinean al inicio del eje

    • flex-end 🠖 los elementos se alinean al final del eje

    • center 🠖 los elementos se alinean al centro del eje

align-content

Si tenemos un contenedor de una sola línea (donde flex-flow se establece como no-wrap) utilizaremos align-items, pero en el caso de que estemos trabajando con contenedores multilínea debemos utilizar align-content.

Esta propiedad nos sirve para alinear items sobre el cross axis cuando los contenedores flexibles incluyen varias lineas (flex-flow es wrap o wrap-reverse)

Estructura básica

xxxxxxxxxx
.flex-container {
    display: flex;
}

flex-wrap

Por defecto los flex-items van a tratar de entrar todos en una misma línea:

Si queremos que se respete el ancho definido de sus hijos tenemos que usar la propiedad flex-wrap: wrap; con el padre:

flex-wrap puede tomar otros valores como nowrap (esta todo en una línea) o wrap-reverse (intercambia cross-start y cross-end).

Ítems

Podemos aplicarle propiedades directamente a cada ítem para manipularlos por separado y tener mayor control.

order

Nos sirve para controlar el orden de cada ítem, sin importar la estructura original del HTML. Por defecto todos los flex-items tienen un order: 0; pero podemos asignarle nosotros otro numero (ya sea positivo o negativo).

  • order > 0 🠖 el ítem pasa al final de la fila o columna (siempre y cuando el resto tengan el valor por defecto 0)

  • order < 0 🠖 el ítem pasa al principio de la fila o columna (siempre y cuando el resto tengan el valor por defecto 0)

Las cajas se irán ordenando respetando la secuencia de números negativos a positivos.

flex-grow

Nos permite definir qué tanto puede llegar a crecer un ítem en caso de disponer espacio libre en el contenedor. Los valores que puede tomar van de 0 a 1, donde representan el 0% y el 100% respectivamente.

Si todos los ítems tienen flex-grow: 1; a medida que el contenedor se agrande, irán abarcando el espacio disponible en partes iguales.

Si lo tiene sólo una caja solo crece esa:

align-self

Nos permite alinear sobre el cross axis a cada ítem al que le apliquemos esta propiedad, independientemente de la alineación definida en el contenedor flex.

Los valores que puede tomar son flex-start, flex-end, flex-center o flex-stretch,

Grid

CSS grid es un modelo de maquetado basado en un sistema de rejillas incorporado en CSS3 para gestionar de manera sencilla y customizable la disposición de nuestros elementos en nuestra página web.

Cómo funciona

Para activar la cuadrícula grid hay que utilizar sobre el elemento contenedor la propiedad display y especificar el valor grid o inline-grid. Este valor influye en cómo se comportará la cuadrícula con el contenido exterior.

  • inline-grid → establece una cuadrícula con ítems en línea, de forma equivalente a inline-block
  • grid → establece una cuadrícula con ítems en bloque, de forma equivalente a block
xxxxxxxxxx
.grid {
    display: grid;
}

Es posible crear cuadrículas con un tamaño explícito usando las propiedades CSS grid-template-columns y grid-template-rows, que sirven para indicar las dimensiones de cada celda de la cuadrícula. Por ejemplo:

xxxxxxxxxx
.grid {
    display: grid;
    grid-template-columns: 1fr 2fr 1fr 1fr 1fr;
    grid-template-rows: 100px 100px;
    grid-gap: 5px;
}

En algunos casos, en las propiedades grid-template-columns y grid-template-rows podemos necesitar indicar las mismas cantidades un número alto de veces, y para no repetir tantas veces lo mismo podemos usar el repeat([num veces], [valor(es)]). Por ejemplo:

xxxxxxxxxx
grid-template-columns: 100px repeat(2, 50px) 200px;
grid-template-rows: repeat(2, 50px 100px);

Ubicar los elementos

Después de definir la rejilla, se colocan en ella los elementos. Para esto, debemos crear los elementos y especificar los valores de inicio y fin. No obstante, no todos los elementos tienen que ocupar necesariamente una sola celda dentro de la cuadrícula.

xxxxxxxxxx
<div class="grid-container">
    <div class="grid-item1">1</div>
    <div class="grid-item2">2</div>
    <div class="grid-item2">3</div>
    <div class="grid-item2">4</div>
    <div class="grid-item2">5</div>
    <div class="grid-item2">6</div>
</div>
x
.grid-container {
    display: grid;
    grid-template-rows: 100px 100px 100px 100px;
    grid-template-columns: 100px 100px 100px 100px;
    grid-gap: 5px;
}
.grid-item1 { 
    background: #99CCFF;
    text-align: center; 
    border: black 5px solid;
    grid-column-start: 1;
    grid-column-end: 5;
    grid-row-start: 1;
    grid-row-end: 3;
}
.grid-item2 { 
    background: #FF9999; 
    text-align: center; border: black 5px solid; 
}

Otras maneras de escribir lo mismo:

Asignar áreas

CSS grid layout permite combinar celdas en áreas y nombrarlas. Esto facilita la tarea de dividir los elementos en la rejilla. Los ajustes para esto se hacen en el contenedor con grid-template-areas.

xxxxxxxxxx
.grid-container {
     display: grid;
     grid-template-rows: 100px 100px 100px;
     grid-template-columns: 1fr 1fr 1fr 1fr 1fr;
     grid-gap: 5px;
     grid-template-areas:
         "area1 area1 area1 area1 area1"
         "area2 area2 . area3 area3"
         "area2 area2 area4 area4 area4";
}

Para definir los elementos, ya no será necesario especificar los valores desde-hasta. Basta con referenciar el área correspondiente.

xxxxxxxxxx
.grid-item1 {
    background: blue;
    text-align: center;
    border: black 5px solid;
    grid-area: area1;
}
.grid-item2 {
    background: red;
    text-align: center;
    border: black 5px solid;
    grid-area: area2;
}
.grid-item3 {
    background: green;
    text-align: center;
    border: black 5px solid;
    grid-area: area3;
}
.grid-item4 {
    background: yellow;
    text-align: center;
    border: black 5px solid;
    grid-area: area4;
}

Posiciones

align-self

Con align-self podemos alinear un elemento de la cuadrícula dentro de una celda a lo largo del eje del bloque (columna). Este valor se aplica al contenido dentro de un solo elemento de cuadrícula.

justify-self

Con justify-self alineamos un elemento de la cuadrícula dentro de una celda a lo largo del eje en línea (fila). Este valor se aplica a un elemento de la cuadrícula dentro de una sola celda.

justify-content

A veces, el tamaño total de su cuadrícula puede ser menor que el tamaño de su contenedor de cuadrícula. Esto podría suceder si todos los elementos de su cuadrícula tienen el tamaño de unidades no flexibles como px. En este caso, puede establecer la alineación de la cuadrícula dentro del contenedor de la cuadrícula. Esta propiedad alinea la cuadrícula a lo largo del eje en línea (fila).

place-self

Con place-self: valor; se establecen las propiedades align-self y justify-self en una sola declaración. Estas son sólo algunas combinaciones:

  • place-self: stretch center;

  • place-self: center start;

  • place-self: start end;

    place-self: end center;
